# Reference

Reference is **the manual page** — terse, accurate, exhaustive. Reach for it when you already know what you're looking for and want the precise details.

## Sections

1. **Routes & screens** — every public URL on the web dashboard, what each one does, who it's for.
2. **Permissions** — every Android permission we declare, what we use it for, what happens if you decline.
3. **Firestore data model** — every collection, every field, every index used by the user-facing layer.
4. **Service modules + analytics events** — the front-end module map and the catalog of telemetry events.
5. **Configuration & environment variables** — every `VITE_*` key, what it does, where to set it.
6. **Quotas, limits, fair-use thresholds** — guest quota, 10-device cap, batch size limits, per-device rates.
7. **Firestore security rules** — high-level summary of who can read/write what.
